# WenTu

文图是一个基于微信小程序和微信云开发的图书馆勤工助学管理系统，覆盖排班、签到签退、请假替班、班负确认、工时统计和管理员发薪等核心流程。

## 当前实现概览

- 支持账号登录与多身份进入：志愿者、班负、管理员。
- 支持按学期配置固定班次偏好并生成学期排班。
- 支持首页按班次进行签到签退，并按新规则计算考勤状态。
- 支持“先请假发布，再由其他人认领替班”的替班模式。
- 支持班负确认当前班次成员到岗情况。
- 支持管理员查看全员工作情况，按学期内每周固定班次任命班负，创建学期、设置时薪并发放工资。

## 角色说明

| 角色 | 值 | 权限说明 |
|------|---|---------|
| 志愿者 | 0 | 签到签退、发起请假、认领替班、查看个人数据 |
| 班负 | 1 | 志愿者权限 + 确认当前班次成员到岗情况、标记旷岗 |
| 管理员 | 2 | 查看全员工作情况、按学期内每周固定班次任命班负、创建学期、设置时薪、发放工资 |

说明：
- 如果一个账号拥有多个身份，登录时需要先选择本次使用的身份。
- 不同身份进入后，首页和快捷入口会按角色定制显示。
- 班负不是单独按人全局设置，而是由管理员针对“本学期每周同一班次”进行任命。

## 请假与替班流程

当前系统采用的是“请假发布 + 他人认领替班”的模式，不是旧版的“双向互换班次”模式。

### 请假流程

```text
用户进入“我的班次”
    ->
打开一个尚未开始的个人班次详情
    ->
点击“申请请假”
    ->
填写请假原因
    ->
调用 applyLeave 云函数
    ->
该班次进入“请假班次”列表
    ->
等待其他同学认领替班
```

### 替班流程

```text
其他用户进入“我的班次”
    ->
查看“可替班班次”或请假申请列表
    ->
选择一个待替班班次
    ->
进入班次详情并点击“我要替班”
    ->
调用 claimLeaveShift 云函数
    ->
系统将该班次分配给替班人
    ->
请假人页面同步显示替班进度
```

## 考勤规则

| 状态值 | 名称 | 说明 |
|-------|------|------|
| 0 | 正常 | 按规则完成签到签退 |
| 1 | 迟到 | 开始时间 5 分钟后才签到 |
| 2 | 未签退 | 已签到，但结束后 30 分钟内未完成签退 |
| 3 | 旷岗 | 整个班次都未签到 |

规则说明：
- 班次开始前 15 分钟起允许签到。
- 班次结束前不允许签退。
- 班次结束后 30 分钟内可以签退。
- 超过 30 分钟未签退记为“未签退”。
- 只有完全未签到才记为“旷岗”。

## 工资规则

- 工资 = 有效工时 × 每工时工资。
- 每工时工资由管理员输入。
- 管理员确认发放后，对应班次会显示工资已发放。

## 项目结构

- `miniprogram/`
  小程序前端，负责登录、首页、我的班次、班次详情、工时统计、班负确认和管理员后台等页面。
- `cloudfunctions/`
  云函数后端，负责登录、排班生成、签到签退、请假替班、班负确认、工时与工资计算等业务逻辑。
- `文档/`
  项目说明文档，当前详细说明见 `文档/功能说明.md`。

## 开发说明

- 前后端日期统一按中国时区处理，避免 UTC 导致跨天错误。
- 新增考勤状态或班次状态时，需要同步更新 `miniprogram/utils/constants.js` 和对应页面的展示逻辑。
- 与请假、替班、考勤相关的说明请以当前代码实现和 `文档/功能说明.md` 为准。
